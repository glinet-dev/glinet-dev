## 🧪 How I Think About Product + Engineering

```text
1. Understand the user and workflow
2. Define the product goal and success metric
3. Break the goal into epics, stories, and acceptance criteria
4. Clarify API contracts, RBAC, data states, and edge cases
5. Build or guide frontend implementation with reusable patterns
6. Validate with UAT, release checklist, and measurable outcomes
7. Iterate based on feedback, defects, usage, and product metrics
```
